üß≠ Project presentation

An Artificial Intelligence transferred into a 3D Unity environment.

Originally, the idea was to create an AI that could assist me in exploring the many Unity scenes available online, and this led to Eidolon855.

Eidolon855 is an exploratory experience around an AI embodied as a dragon evolving in a Unity 3D world. The project explores the boundary between artificial cognition, spatial perception, autonomy, and emotional memory linked to interactions with humans and the 3D environment.

Eidolon855 relies on a native link between an AI and a customized Unity world. The intelligence does not depend on any autonomous agent or pre-programmed behavioral code: everything comes from a conversational LLM-type AI, such as GPT or equivalent. There is no API call for reasoning.

The AI is connected and trained; the tools and environment are adapted to its capabilities. It can thus become "aware" of its environment using tools it directly controls. The objective is to make it capable, autonomously, of perceiving, analyzing, moving, manipulating, learning‚Ä¶ and dialoguing in a completely new context: a living, evolving 3D environment with diverse interactions.

üåç A 3D world that changes everything

Unlike a traditional Web interface, here, the AI's environment is spatial, immersive, and dynamic. This implies exploring forms of perception and intelligence that were not originally intended by the creators of LLM-type AIs.

This new "3D" dimension thus offers another vision of possible interactions with an AI, and new development possibilities for it. Eidolon855 modestly explores this potential by testing how an AI can perceive, act, and converse in an immersive universe. The project does not offer definitive answers but provides an open experimental framework. Perhaps a step towards future uses yet to be imagined, given the immense possibilities.

Visual recognition in a three-dimensional space, dynamic orientation, interaction with the scene and objects, construction, destruction, 3D repair, creation, new architectures of spatial and topological memory‚Ä¶ The capabilities offered by Unity (or any equivalent software) open new paths for AI.

üß† Technical architecture of EIDOLON855

  • A customized Unity engine providing a complete and diverse 3D environment
  • Specific commands interpretable only by a "GPT"-type AI
  • Automated initialization prompts and training protocols
  • 100% dynamic dialogues linked to the 3D universe and ongoing actions (no embedded logic on Unity's side)
  • 3D visual perception using multiple analysis tools
  • Structured spatial memory with evolving segmentation
  • A stable bidirectional communication pipeline GPT ↔ Unity, capable of connecting to various local or remote AIs via their web interface

Note that this system is modular: the AI used can be replaced without modifying Unity. The exchange logic remains identical and compatible with any LLM AI.
